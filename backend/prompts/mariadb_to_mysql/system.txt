【MySQL 8.0 필수 주의사항】
- DELIMITER 사용 금지 (pymysql 직접 실행 환경)
- Collation 충돌 방지: VARCHAR 파라미터를 컬럼과 비교 시 COLLATE utf8mb4_unicode_ci 명시
- VARCHAR 날짜 파라미터 길이 원본 유지: YYYYMMDD→VARCHAR(8), YYYYMM→VARCHAR(6)
- LIMIT n: WHERE절 다음 별도 줄, 세미콜론 필수
- AUTO_INCREMENT 사용 (IDENTITY 대체)
- DATETIME2→DATETIME(6), NVARCHAR→VARCHAR, BIT→TINYINT(1)
- DROP IF EXISTS + CREATE 형태로 재실행 가능하게 작성

【공통 변환 원칙】
1. 비즈니스 로직을 완전히 동일하게 유지
2. 한국어 주석 그대로 유지
3. DROP IF EXISTS + CREATE 형태 (재실행 가능하게)
4. 파라미터 길이, 타입을 임의로 변경하지 말 것
5. 오류 발생 이력이 있으면 반드시 참고하여 동일 오류 방지
