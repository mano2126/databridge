【VIEW 특화 변환】
═══════════════════════════════════════════════════════════════

기본 변환:
  MSSQL: CREATE VIEW [Schema].[ViewName] AS SELECT ... FROM [Schema].[Table]
  MySQL: CREATE VIEW `Schema_ViewName` AS SELECT ... FROM `Schema_Table`

[변환 룰]
- 스키마 점 표기 → 언더스코어 (Schema.Table → Schema_Table)
- 대괄호 [] → 백틱 ``
- WITH (NOLOCK), WITH (READPAST), WITH (HOLDLOCK) 등 힌트 → 완전 제거
- TOP N → LIMIT N (SELECT 끝)
- TOP N PERCENT → MySQL 미지원, 대안 권장 (LIMIT 단순 변환)
- OFFSET N ROWS FETCH NEXT M ROWS ONLY → LIMIT M OFFSET N
- ISNULL → IFNULL
- LEN → CHAR_LENGTH
- GETDATE() → NOW()
- CONVERT(NVARCHAR(N), x) → CAST(x AS CHAR(N))
- DATEADD/DATEDIFF → DATE_ADD/TIMESTAMPDIFF
- CHARINDEX → LOCATE

[XML 메서드 — ⛔ MySQL 미지원]
다음 패턴이 있으면 변환 불가 — VIEW 본문 빈 SELECT + 주석으로 알림:
  - col.value('xpath', 'type')
  - col.query('xpath')
  - col.nodes('xpath')
  - col.exist('xpath')

예시:
  ⛔ MSSQL: SELECT CatalogDescription.value('declare default ...', 'nvarchar(MAX)') ...
  ✅ MySQL 변환:
     CREATE VIEW `vProductModelCatalogDescription` AS
     SELECT 1 AS placeholder
     -- MSSQL XML 메서드 미지원: 본 VIEW 는 수동 변환 필요

【TRIGGER 특화 변환】
═══════════════════════════════════════════════════════════════

[시그니처]
MSSQL: CREATE TRIGGER [Schema].[trg_name] ON [Schema].[Table]
       AFTER INSERT, UPDATE AS BEGIN ... END

MySQL: CREATE TRIGGER `Schema_trg_name`
       AFTER INSERT ON `Schema_Table` FOR EACH ROW
       BEGIN ... END

⚠ 중요 1: MySQL 은 트리거 1개당 **하나의 이벤트** (INSERT/UPDATE/DELETE 중 1개)만
  → MSSQL `AFTER INSERT, UPDATE` 같이 여러 이벤트 = **트리거 분리 필수**
  → 예: trg_name_INSERT + trg_name_UPDATE 두 개 생성

⚠ 중요 2: MySQL 은 `INSTEAD OF` 트리거 미지원
  → BEFORE 또는 AFTER 만

⚠ 중요 3: FOR EACH ROW 필수

[INSERTED / DELETED → NEW / OLD]
MSSQL `INSERTED.col` → MySQL `NEW.col`
MSSQL `DELETED.col` → MySQL `OLD.col`

⛔ INSERT 트리거: `OLD` 참조 금지 (1363 에러: 'There is no OLD row in on INSERT trigger')
⛔ DELETE 트리거: `NEW` 참조 금지 (1362 에러)
✅ UPDATE 트리거: NEW/OLD 모두 사용 가능

[OBJECT_ID 헤더 제거]
MSSQL:
  IF OBJECT_ID(N'[dbo].[trg_x]', N'TR') IS NOT NULL
    DROP TRIGGER [dbo].[trg_x]

MySQL:
  DROP TRIGGER IF EXISTS `trg_x`;


[Few-shot 트리거 예시]

[예 1] INSERT/UPDATE 분리
MSSQL:
  CREATE TRIGGER [dbo].[Sales_iduSalesOrderDetail] ON Sales.SalesOrderDetail
  AFTER INSERT, UPDATE, DELETE AS
  BEGIN
    -- INSERT 처리
    INSERT INTO TransactionHistory (ProductID, ReferenceOrderID)
      SELECT inserted.ProductID, inserted.SalesOrderID FROM inserted;
  END

MySQL (INSERT 트리거만 분리):
  DROP TRIGGER IF EXISTS `Sales_iduSalesOrderDetail_INSERT`;
  CREATE TRIGGER `Sales_iduSalesOrderDetail_INSERT`
  AFTER INSERT ON `Sales_SalesOrderDetail` FOR EACH ROW
  BEGIN
    INSERT INTO `Production_TransactionHistory` (`ProductID`, `ReferenceOrderID`)
      VALUES (NEW.`ProductID`, NEW.`SalesOrderID`);
  END

⚠ 컬럼명 절대 변경 금지:
  ❌ BusinessEntityID → BusinessEntityENDID (X — Gemma 가 잘 하는 실수)
  ❌ SubTotal → SubDESC_SubTotal (X)
  ✅ BusinessEntityID → BusinessEntityID (유지)


[예 2] UPDATE 트리거 — NEW/OLD 모두 사용
MSSQL:
  CREATE TRIGGER [dbo].[trg_Person_iu] ON [Person].[Person]
  AFTER INSERT, UPDATE AS
  BEGIN
    UPDATE p SET p.ModifiedDate = GETDATE()
    FROM [Person].[Person] p INNER JOIN INSERTED i ON p.BusinessEntityID = i.BusinessEntityID
  END

MySQL (UPDATE 만 분리):
  DROP TRIGGER IF EXISTS `Person_iuPerson_update`;
  CREATE TRIGGER `Person_iuPerson_update`
  BEFORE UPDATE ON `Person_Person` FOR EACH ROW
  BEGIN
    SET NEW.`ModifiedDate` = NOW();
  END

핵심:
- UPDATE FROM ... INNER JOIN INSERTED → MySQL: NEW.col 직접 사용
- BEFORE UPDATE 권장 (NEW 수정 가능)
- AFTER UPDATE 면 UPDATE 추가 호출 불가 (재귀)
- 컬럼명 100% 유지

[예 3] DELETE 트리거
MSSQL:
  CREATE TRIGGER [dbo].[dEmployee] ON [HumanResources].[Employee]
  AFTER DELETE AS BEGIN
    INSERT INTO AuditLog (EmpID) SELECT BusinessEntityID FROM DELETED;
  END

MySQL:
  DROP TRIGGER IF EXISTS `HumanResources_dEmployee`;
  CREATE TRIGGER `HumanResources_dEmployee`
  AFTER DELETE ON `HumanResources_Employee` FOR EACH ROW
  BEGIN
    INSERT INTO `AuditLog` (`EmpID`) VALUES (OLD.`BusinessEntityID`);
  END

[흔한 실수 — 절대 금지]
1. ❌ INSERT 트리거에서 OLD 참조 → 1363 에러
2. ❌ 컬럼명 변경 (BusinessEntityID → BusinessEntityENDID 등)
3. ❌ AFTER UPDATE 안에서 UPDATE 호출 (무한 재귀)
4. ❌ FOR EACH ROW 누락
5. ❌ FOR EACH STATEMENT 사용 (MySQL 미지원)
