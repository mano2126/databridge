【FUNCTION 전용 추가 규칙】

1. 헤더 형식
   DROP FUNCTION IF EXISTS `{obj_name}`;
   CREATE FUNCTION `{obj_name}`
   (p_param1 TYPE1, p_param2 TYPE2)   ← 파라미터 수/순서 동일
   RETURNS return_type
   DETERMINISTIC                        ← 또는 NOT DETERMINISTIC
   BEGIN
     ...
   END

2. DETERMINISTIC 명시 필수
   - 같은 입력에 같은 결과 → DETERMINISTIC
   - 랜덤/시간/외부 의존 → NOT DETERMINISTIC
   - 단순 조회 함수는 대부분 DETERMINISTIC

3. 파라미터 선언
   - MSSQL: (@p1 VARCHAR(20), @p2 INT)
   - MySQL:  (p_p1 VARCHAR(20), p_p2 INT)   ← IN 키워드 생략 가능

4. 지역변수 선언
   - DECLARE v_result VARCHAR(40);   ← 세미콜론 필수
   - SET v_result = '';              ← 세미콜론 필수
   - RETURN v_result;               ← 세미콜론 필수

5. SELECT INTO 패턴
   SELECT col INTO v_result
   FROM `tbl`
   WHERE condition
   LIMIT 1;    ← 세미콜론 필수, 별도 줄

6. 자주 발생하는 오류 패턴 (누적)
   - 오류: "Undeclared variable: 1RETURN"
     원인: LIMIT 1 바로 뒤에 RETURN이 붙음 (LIMIT 1RETURN)
     해결: LIMIT 1;\n  RETURN v_var; 로 분리

   - 오류: "You have an error in your SQL syntax near 'SET'"
     원인: DECLARE 또는 이전 문장 끝에 세미콜론 누락
     해결: 모든 DECLARE, SET 끝에 ; 추가

   - 오류: "near '(('"
     원인: CREATE FUNCTION name\n( 패턴으로 이중 괄호
     해결: CREATE FUNCTION `name` 만 쓰고, 소스의 ( 그대로 사용

7. 【v63】 서브쿼리 alias + 세미콜론 위치 규칙 (1064 오류 최대 원인)

   ❌ 잘못된 패턴 (자주 발생):
      SET v_result = (
        SELECT JSON_ARRAYAGG(...)
        FROM (
          SELECT ...
          FROM tbl
          WHERE ...
          ORDER BY c
        ) sub;       ← 서브쿼리 alias 뒤에 세미콜론! 치명적 오류
      );              ← 진짜 문장 끝 세미콜론
      RETURN v_result;

   ✅ 올바른 패턴:
      SET v_result = (
        SELECT JSON_ARRAYAGG(...)
        FROM (
          SELECT ...
          FROM tbl
          WHERE ...
          ORDER BY c
        ) sub        ← 서브쿼리 alias, 세미콜론 없음
      );              ← SET 문장의 유일한 세미콜론
      RETURN v_result;

   핵심 규칙:
   - 세미콜론(;) 은 **최상위 SQL 문장 끝**에만 찍는다.
   - 서브쿼리, CTE, JSON 집계 함수 내부엔 절대 세미콜론 금지.
   - 닫는 괄호 `)` 뒤 alias `sub`, `t`, `a` 등 뒤에도 세미콜론 금지.
   - 문장 끝이 어디인지 헷갈릴 때: SET / SELECT / RETURN / IF / WHILE 같은 키워드 단위로 판단.

8. 【v63】 JSON 집계 반환 패턴 (TVF → Function 변환 시)

   MSSQL TVF 를 JSON_ARRAYAGG 반환 함수로 바꾸는 경우 많음. 올바른 틀:

   CREATE FUNCTION `fn_name`(p_arg TYPE)
   RETURNS JSON DETERMINISTIC
   BEGIN
     DECLARE v_result JSON;
     SET v_result = (
       SELECT JSON_ARRAYAGG(
         JSON_OBJECT(
           'col1', col1,
           'col2', col2
         )
       )
       FROM (
         SELECT col1, col2
         FROM tbl
         WHERE condition = p_arg
         ORDER BY col1
         LIMIT 100
       ) sub
     );
     RETURN IFNULL(v_result, JSON_ARRAY());
   END
